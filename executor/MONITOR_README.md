# TestFlow 执行引擎监控GUI

## 📱 简介

这是一个纯监控界面，用于实时查看TestFlow执行引擎的状态和日志，**不包含任何操作功能**。

## ✨ 主要功能

### 📊 状态监控
- **引擎状态**: 显示执行引擎是否正在运行
- **后端连接**: 显示与后端API的连接状态
- **活动任务**: 当前正在执行的任务数量
- **已完成任务**: 已完成的任务总数
- **队列大小**: 等待执行的任务数量
- **连接设备**: 已连接的Android设备数量
- **运行时间**: 引擎运行时间

### 📋 任务列表
- 实时显示所有活动任务
- 显示任务ID、类型、目标ID、设备、状态和进度
- 自动刷新任务状态

### 📱 设备监控
- 显示所有连接的Android设备
- 显示设备序列号、连接状态

### 📋 日志查看
- 实时显示系统日志
- 彩色标记不同级别的日志（INFO/WARNING/ERROR/SUCCESS）
- 自动滚动到最新日志
- 限制显示最后50条日志，避免内存占用

## 🚀 使用方法

### 步骤1: 启动执行引擎

```bash
cd C:\Users\lsea.yu\Desktop\docs\testflow\executor
python main.py
```

执行引擎启动后会：
1. 连接到数据库
2. 连接到后端WebSocket (ws://localhost:8000/ws/executor)
3. 启动HTTP服务器 (http://127.0.0.1:5555) ← 监控GUI连接到这个
4. 开始轮询数据库中的待执行任务

### 步骤2: 启动监控GUI

**方式1: 使用批处理文件（推荐）**
```bash
start_monitor.bat
```

**方式2: 直接运行Python脚本**
```bash
python monitor.py
```

**方式3: 指定自定义URL**
```bash
python monitor.py http://192.168.1.100:5555
```

## 🔧 工作原理

```
┌─────────────────┐
│  监控GUI (Tkinter) │
│   只读界面        │
└────────┬─────────┘
         │ HTTP GET请求
         ↓ (每3秒轮询)
┌─────────────────┐
│ 执行引擎本体      │
│  - HTTP服务器    │ ← 监听 http://127.0.0.1:5555
│  - 执行逻辑      │
│  - WebSocket客户端│ → 连接到后端API
└────────┬─────────┘
         │ WebSocket
         ↓
┌─────────────────┐
│  后端API服务器   │
│  - 任务管理      │
│  - WebSocket服务 │
└─────────────────┘
```

## ⚙️ 配置选项

### 修改刷新频率

在 `monitor.py` 中修改：

```python
# 默认3秒刷新一次
self.root.after(3000, self.auto_refresh)

# 改为5秒刷新
self.root.after(5000, self.auto_refresh)
```

### 修改HTTP服务器地址

在 `main.py` 中修改HTTP服务器的监听地址：

```python
self.http_server = ExecutorHTTPServer(
    self.executor,
    host='0.0.0.0',  # 允许远程连接
    port=5555
)
```

### 修改日志显示数量

在 `monitor.py` 中修改：

```python
# 默认显示最后50条
for log in new_logs[-50:]:
    # 改为显示最后100条
    for log in new_logs[-100:]:
```

## 🛠️ 故障排除

### 问题1: 监控界面显示"连接断开"

**原因**: 执行引擎未启动或HTTP服务器未启动

**解决方案**:
1. 检查执行引擎是否正在运行：`python main.py`
2. 查看执行引擎日志，确认HTTP服务器已启动
3. 检查端口5555是否被占用

### 问题2: 状态不更新

**原因**: HTTP请求失败或超时

**解决方案**:
1. 检查网络连接
2. 确认监控GUI连接到正确的URL
3. 查看执行引擎日志

### 问题3: 日志显示异常

**原因**: 日志文件读取失败

**解决方案**:
1. 确认 `executor.log` 文件存在
2. 检查文件权限
3. 查看执行引擎日志输出

## 🔒 安全说明

- HTTP服务器默认只监听 `127.0.0.1`，只能本地访问
- 监控界面没有操作功能，纯只读访问
- 不包含任何认证机制，仅用于开发环境
