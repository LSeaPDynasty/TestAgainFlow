# Code Review Skill

严格审查代码质量、安全性和最佳实践，提供专业级代码审查报告。

## 使用场景

- 审查 Pull Request 代码变更
- 评估项目整体代码质量
- 发现潜在安全漏洞
- 检查代码规范和最佳实践
- 提供重构和优化建议

## 审查维度

### 1. 安全性 (Security)
- [ ] 硬编码密钥、密码、Token
- [ ] SQL注入、XSS、CSRF漏洞
- [ ] 不安全的随机数生成
- [ ] 敏感信息日志输出
- [ ] 认证授权缺陷
- [ ] CORS配置不当
- [ ] 依赖库已知漏洞

### 2. 代码质量 (Code Quality)
- [ ] 异常处理是否完善
- [ ] 是否有裸except子句
- [ ] 代码重复率
- [ ] 函数/类复杂度
- [ ] 命名规范
- [ ] 类型注解完整性
- [ ] 注释质量

### 3. 性能 (Performance)
- [ ] 数据库查询优化（N+1问题）
- [ ] 缓存使用
- [ ] 循环效率
- [ ] 内存泄漏风险
- [ ] 并发处理
- [ ] 资源释放

### 4. 可维护性 (Maintainability)
- [ ] 代码结构清晰度
- [ ] 模块耦合度
- [ ] 单一职责原则
- [ ] 代码可读性
- [ ] 文档完整性

### 5. 最佳实践 (Best Practices)
- [ ] 框架使用规范
- [ ] 设计模式应用
- [ ] 错误处理模式
- [ ] 日志记录规范
- [ ] 测试覆盖

### 6. Python特定
- [ ] 使用logging而非print
- [ ] 依赖注入
- [ ] 异步/同步混用
- [ ] GIL相关问题
- [ ] 依赖版本管理

### 7. TypeScript/React特定
- [ ] 类型安全
- [ ] Props验证
- [ ] 状态管理
- [ ] 性能优化（memo, callback）
- [ ] 内存泄漏（useEffect cleanup）

### 8. 安全配置
- [ ] CORS配置
- [ ] 认证中间件
- [ ] 速率限制
- [ ] 输入验证
- [ ] 输出编码

## 审查流程

1. **理解上下文**: 了解项目类型、技术栈、业务场景
2. **逐项检查**: 按照上述维度系统性检查
3. **问题分类**: 将问题分为 P0(严重) / P1(重要) / P2(建议)
4. **提供示例**: 展示问题代码和修复方案
5. **评分总结**: 给出总体评分和改进建议

## 示例命令

```bash
# 审查单个文件
/review path/to/file.py

# 审查整个目录
/review backend/

# 审查特定变更
/review --diff main.py

# 按维度审查
/review --security backend/app/
/review --performance frontend/src/

# 生成详细报告
/review --detailed backend/app/models/
```

## 注意事项

- 审查要具体，提供代码示例
- 区分"错误"和"风格偏好"
- 考虑项目实际需求和约束
- 提供建设性意见，而非批评
- 对AI生成的代码要特别警惕过度设计
