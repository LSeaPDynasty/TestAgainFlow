
### 层次说明

| 层级 | 资源类型 | 说明 | 复用性 |
|------|---------|------|--------|
| 1 | **Element (元素)** | UI元素的抽象表示，包含多种定位符 | ⭐⭐⭐⭐⭐ 最高复用 |
| 2 | **Step (步骤)** | 原子操作，引用元素定义具体行为 | ⭐⭐⭐⭐ 高复用 |
| 3 | **Flow (流程)** | 步骤序列，可嵌套子流程 | ⭐⭐⭐ 中复用 |
| 4 | **Testcase (用例)** | 完整测试场景，组合多个流程 | ⭐⭐ 低复用 |
| 5 | **Suite (套件)** | 批量用例集合，用于批量执行 | ⭐ 一次性组合 |

## 数据模型关系

### 1. Element (元素) - 最基础的复用单元

#### 结构定义
```python
class Element:
    id: int                          # 元素ID
    name: str                        # 元素名称 (全局唯一)
    screen_id: int                   # 所属界面
    description: str                 # 描述
    locators: List[Locator]          # 定位符数组 (优先级排序)

class Locator:
    type: str                        # 定位符类型
    value: str                       # 定位符值
    priority: int                    # 优先级 (1=最高)
```

#### 设计理念
**多定位符策略** - 提高元素查找的鲁棒性

```json
{
  "name": "login_button",
  "screen": "登录页",
  "locators": [
    {
      "type": "resource-id",
      "value": "com.app:id/btn_login",
      "priority": 1
    },
    {
      "type": "xpath",
      "value": "//*[@text='登录']",
      "priority": 2
    },
    {
      "type": "text",
      "value": "登录",
      "priority": 3
    }
  ]
}
```

**查找逻辑**:
```
1. 尝试 priority=1 的定位符
   └─ 成功 → 返回元素
   └─ 失败 → 继续

2. 尝试 priority=2 的定位符
   └─ 成功 → 返回元素
   └─ 失败 → 继续

3. 尝试 priority=3 的定位符
   └─ 成功 → 返回元素
   └─ 失败 → 返回 null
```

### 2. Step (步骤) - 原子操作

#### 结构定义
```python
class Step:
    id: int                          # 步骤ID
    name: str                        # 步骤名称
    screen_id: int                   # 所属界面
    action_type: str                 # 操作类型
    element_id: int?                 # 目标元素 (可选)
    flow_id: int?                    # 目标流程 (用于调用子流程)
    action_value: str?               # 操作值 (支持变量 {{username}})
    assert_config: dict?             # 断言配置
    wait_after_ms: int               # 等待时间 (毫秒)
    continue_on_error: bool          # 失败是否继续
```

#### 操作类型分类

**设备操作类**
```json
{
  "action_type": "click",
  "element": "login_button",
  "wait_after_ms": 300
}

{
  "action_type": "input",
  "element": "username_input",
  "action_value": "{{username}}",
  "wait_after_ms": 200
}

{
  "action_type": "swipe",
  "action_value": "up",
  "duration": 500
}
```

**等待操作类**
```json
{
  "action_type": "wait_element",
  "element": "home_tab",
  "timeout": 30
}

{
  "action_type": "wait_time",
  "action_value": "2000"  // 毫秒
}
```

**断言操作类**
```json
{
  "action_type": "assert_exists",
  "element": "success_message",
  "assert_config": {
    "type": "exists",
    "expected": "true",
    "on_fail": "stop"
  }
}

{
  "action_type": "assert_text",
  "element": "title_text",
  "action_value": "欢迎",
  "assert_config": {
    "type": "equals",
    "expected": "欢迎",
    "on_fail": "continue"
  }
}
```

**流程调用类**
```json
{
  "action_type": "call_flow",
  "flow": "common_login_flow",
  "params": {
    "username": "{{test_user}}",
    "password": "{{test_password}}"
  }
}
```

### 3. Flow (流程) - 步骤序列编排

#### 结构定义
```python
class Flow:
    id: int                          # 流程ID
    name: str                        # 流程名称 (全局唯一)
    description: str                 # 描述
    flow_type: str                   # standard / dsl / python
    project_id: int?                 # 所属项目

class FlowStep:                      # 流程-步骤关联表
    flow_id: int                     # 父流程ID
    step_id: int?                    # 引用的步骤ID
    sub_flow_id: int?                # 引用的子流程ID
    order_index: int                 # 执行顺序
    override_value: str?             # 覆盖步骤的action_value
```

#### FlowStep 的两种类型

**类型1: 引用步骤 (step_id 不为空)**
```json
{
  "flow_id": 1,
  "step_id": 10,
  "order_index": 1,
  "override_value": null
}
```

**类型2: 嵌套子流程 (sub_flow_id 不为空)**
```json
{
  "flow_id": 1,
  "sub_flow_id": 5,
  "order_index": 2,
  "override_value": null
}
```

#### 流程嵌套示例

```
LoginFlow (登录流程)
├─ Step 1: 输入用户名 (step_id=100)
├─ Step 2: 输入密码 (step_id=101)
└─ Step 3: 点击登录 (step_id=102)

MainFlow (主流程)
├─ Step 1: 打开应用 (step_id=1)
├─ FlowStep 2: LoginFlow (sub_flow_id=10)  ← 嵌套登录流程
└─ Step 3: 验证登录成功 (step_id=200)
```

#### 执行顺序保证
```python
# Flow执行时按 order_index 排序
flow_steps = FlowStep.query.filter_by(flow_id=1).order_by('order_index').all()

for flow_step in flow_steps:
    if flow_step.step_id:
        # 执行步骤
        await execute_step(flow_step.step)
    elif flow_step.sub_flow_id:
        # 递归执行子流程
        await execute_flow(flow_step.sub_flow)
```

### 4. Testcase (测试用例) - 完整测试场景

#### 结构定义
```python
class Testcase:
    id: int                          # 用例ID
    name: str                        # 用例名称 (全局唯一)
    description: str                 # 描述
    priority: str                    # P0/P1/P2/P3
    timeout: int                     # 超时时间 (秒)
    params: dict?                    # 用例级参数
    project_id: int?                 # 所属项目

class TestcaseFlow:                  # 用例-流程关联表
    testcase_id: int                 # 用例ID
    flow_id: int                     # 流程ID
    flow_role: str                   # setup / main / teardown
    order_index: int                 # 执行顺序
    enabled: bool                    # 是否启用
    params: dict?                    # 覆盖流程参数
```

#### 三种流程角色

**1. Setup (前置流程)**
- 用例执行前的准备工作
- 失败则整个用例不执行
- 示例: 打开应用、登录、进入测试页面

**2. Main (主流程)**
- 用例的核心测试逻辑
- 可以有多个主流程
- 示例: 执行业务操作、验证功能

**3. Teardown (清理流程)**
- 用例执行后的清理工作
- 无论Main成功或失败都会执行
- 示例: 退出登录、清理数据、关闭应用

#### 用例执行顺序

```
┌─────────────────────────────────────────────────────────┐
│  Testcase Execution Order                               │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. Setup Flows (按 order_index 执行)                   │
│     ┌─────────────────────────────────────┐            │
│     │ Setup Flow 1 → Setup Flow 2 → ...   │            │
│     └─────────────────────────────────────┘            │
│                     ↓                                   │
│              如果 Setup 失败 → 停止执行                 │
│                     ↓                                   │
│              如果 Setup 成功 → 继续                     │
│                     ↓                                   │
│  2. Main Flows (按 order_index 执行)                    │
│     ┌─────────────────────────────────────┐            │
│     │ Main Flow 1 → Main Flow 2 → ...    │            │
│     └─────────────────────────────────────┘            │
│                     ↓                                   │
│              记录 Main 的执行结果                       │
│                     ↓                                   │
│  3. Teardown Flows (无论Main成败都执行)                │
│     ┌─────────────────────────────────────┐            │
│     │ Teardown Flow 1 → Teardown Flow 2 →│            │
│     └─────────────────────────────────────┘            │
│                     ↓                                   │
│              返回最终结果                               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 5. Suite (测试套件) - 批量执行

#### 结构定义
```python
class Suite:
    id: int                          # 套件ID
    name: str                        # 套件名称
    description: str                 # 描述
    parallel_execution: bool         # 是否并行执行

class SuiteTestcase:                # 套件-用例关联表
    suite_id: int                    # 套件ID
    testcase_id: int                 # 用例ID
    order_index: int                 # 执行顺序
    enabled: bool                    # 是否启用
```

#### 套件执行模式

**串行执行 (parallel_execution=false)**
```
Testcase 1
  ↓ (等待完成)
Testcase 2
  ↓ (等待完成)
Testcase 3
  ↓ (等待完成)
...
```

**并行执行 (parallel_execution=true)**
```
Testcase 1 ──┐
Testcase 2 ──┼── 同时执行
Testcase 3 ──┘
  ↓
等待所有完成
```

## 用例执行流程

### 完整执行流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                    Testcase Execution Flow                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 加载用例数据                                                 │
│     ├── testcase.params (用例级参数)                            │
│     ├── testcase.setup_flows[]                                  │
│     ├── testcase.main_flows[]                                   │
│     └── testcase.teardown_flows[]                               │
│                      ↓                                          │
│  2. 初始化执行上下文                                             │
│     ├── 合并参数: testcase.params + flow.params + step.params  │
│     ├── 创建变量作用域                                          │
│     └── 初始化日志记录器                                        │
│                      ↓                                          │
│  3. 执行 Setup 流程                                             │
│     for flow in setup_flows:                                   │
│       result = await execute_flow(flow, context)               │
│       if result == FAILED:                                     │
│         return FAILED  # Setup失败则停止                        │
│                      ↓                                          │
│  4. 执行 Main 流程                                              │
│     main_result = PASSED                                       │
│     for flow in main_flows:                                    │
│       result = await execute_flow(flow, context)               │
│       if result == FAILED:                                     │
│         if not continue_on_error:                             │
│           main_result = FAILED                                │
│           break  # 停止执行剩余Main流程                        │
│                      ↓                                          │
│  5. 执行 Teardown 流程 (无论Main成败)                          │
│     for flow in teardown_flows:                               │
│       await execute_flow(flow, context)  # 忽略结果           │
│                      ↓                                          │
│  6. 生成执行报告                                                 │
│     return {                                                    │
│       "result": main_result,                                   │
│       "setup_results": [...],                                  │
│       "main_results": [...],                                   │
│       "teardown_results": [...],                               │
│       "duration": end_time - start_time,                       │
│       "logs": all_logs                                        │
│     }                                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Flow 执行流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      Flow Execution Flow                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 加载流程数据                                                 │
│     ├── flow.default_params (流程默认参数)                      │
│     └── flow.flow_steps[] (按order_index排序)                   │
│                      ↓                                          │
│  2. 执行流程中的每个 FlowStep                                    │
│     for flow_step in flow_steps:                               │
│                                                                 │
│       if flow_step.step_id:                                    │
│         # ──── 引用步骤 ────                                   │
│         step = load_step(flow_step.step_id)                    │
│         result = await execute_step(step, context)             │
│                                                                 │
│       elif flow_step.sub_flow_id:                              │
│         # ──── 嵌套子流程 ────                                 │
│         sub_flow = load_flow(flow_step.sub_flow_id)            │
│         result = await execute_flow(sub_flow, context)         │
│                                                                 │
│       # 检查结果                                                │
│       if result == FAILED:                                     │
│         if not step.continue_on_error:                        │
│           return FAILED  # 停止执行                            │
│                                                                 │
│     return PASSED                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Step 执行流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      Step Execution Flow                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 解析步骤数据                                                 │
│     ├── step.action_type                                        │
│     ├── step.element (如果需要)                                 │
│     ├── step.action_value (支持变量替换)                        │
│     └── step.assert_config (如果是断言)                          │
│                      ↓                                          │
│  2. 变量替换                                                     │
│     action_value = replace_variables(                          │
│       step.action_value,                                       │
│       context.variables  # {{username}} → "test_user"          │
│     )                                                          │
│                      ↓                                          │
│  3. 查找元素 (如果需要)                                          │
│     if action_type in ["click", "input", "assert_*"]:          │
│       element = find_element_by_locators(                      │
│         step.element.locators                                 │
│       )                                                        │
│       if not element:                                          │
│         return FAILED                                          │
│                      ↓                                          │
│  4. 执行操作                                                     │
│     try:                                                       │
│       if action_type == "click":                               │
│         await driver.click(element)                            │

│         await driver.input(element, action_value)              │
│       elif action_type == "assert_text":                       │
│         actual = await driver.get_text(element)                │
│         if actual != expected:                                │
│           return FAILED                                       │
│       ...                                                      │
│                                                                 │
│     except Exception as e:                                     │
│       log_error(e)                                             │
│       return FAILED                                            │
│                      ↓                                          │
│  5. 等待 (wait_after_ms)                                        │
│     if step.wait_after_ms > 0:                                │
│       await asyncio.sleep(step.wait_after_ms / 1000)           │
│                      ↓                                          │
│  6. 返回结果                                                     │
│     return PASSED                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 设计思路与理念

### 1. 分层复用设计

#### 理念
**小粒度、高复用、低耦合**

#### 实现方式

**Element层 - 最高复用**
```json
{
  "name": "submit_button",
  "locators": [...],
  "reusable_in": [
    "登录流程",
    "注册流程",
    "提交表单流程"
  ]
}
```
- 一个元素可以被成百上千个步骤复用
- 修改元素定位符，所有引用步骤自动生效

**Step层 - 高复用**
```json
{
  "name": "input_username",
  "element": "username_input",
  "action_value": "{{username}}",
  "reusable_in": [
    "登录流程",
    "注册流程",
    "修改用户名"
  ]
}
```
- 一个步骤可以被多个流程复用
- 支持变量参数化，适应不同场景

**Flow层 - 中复用**
```json
{
  "name": "login_flow",
  "steps": [
    "input_username",
    "input_password",
    "click_login"
  ],
  "reusable_in": [
    "需要登录的测试用例"
  ]
}
```
- 流程可以嵌套，形成复杂逻辑
- 业务级别的复用单元

---

### 2. 关联表设计模式

#### 理念
**分离定义与关系，支持多对多和有序组合**

#### FlowStep 关联表

**为什么需要关联表？**
```python
# ❌ 不好的设计 - Flow直接包含steps数组
class Flow:
    steps: List[Step]  # 无法复用step，无法排序

# ✅ 好的设计 - 使用关联表
class Flow:
    flow_steps: List[FlowStep]  # 可以引用step或sub_flow，支持排序
```

**关联表的优势**
```python
class FlowStep:
    flow_id: int          # 父流程
    step_id: int?         # 引用步骤 (可选)
    sub_flow_id: int?     # 引用子流程 (可选)
    order_index: int      # 执行顺序
    override_value: str?  # 覆盖值 (灵活性)
```

**使用场景**
```json
// 场景1: 复用步骤
"login_flow": {
  "flow_steps": [
    {"step_id": 100, "order": 1},  // 输入用户名步骤
    {"step_id": 101, "order": 2},  // 输入密码步骤
    {"step_id": 102, "order": 3}   // 点击登录步骤
  ]
}

// 场景2: 嵌套流程
"main_flow": {
  "flow_steps": [
    {"step_id": 1, "order": 1},        // 打开应用
    {"sub_flow_id": 10, "order": 2},   // 登录流程
    {"step_id": 200, "order": 3}       // 验证登录
  ]
}

// 场景3: 覆盖参数
"login_test_flow": {
  "flow_steps": [
    {"step_id": 100, "order": 1, "override_value": "test_user"},
    {"step_id": 101, "order": 2, "override_value": "test_pass"}
  ]
}
```

### 3. 变量作用域设计

#### 理念
**支持参数传递和覆盖，实现灵活性**

#### 变量优先级
```
1. Step override_value (最高优先级)
   ↓
2. Flow params
   ↓
3. Testcase params
   ↓
4. Testcase global variables
   ↓
5. System defaults (最低优先级)
```

#### 变量替换示例

```python

  "action_value": "{{username}}"
}

# Testcase级别定义变量

    "params": {
      "username": "test_user",
      "password": "test_pass"
    },
    "main_flows": [...]
  }
}

# 执行时替换

            ↓ replace variables

```

#### 参数覆盖链

```

  "username": "default_user"

        ↓

  "username": "flow_user"  # 覆盖testcase的username
}

```

### 4. 错误处理策略

#### 理念
**支持细粒度错误控制，避免一个失败导致全部失败**

#### continue_on_error 设计

```json
{
  "name": "click_optional_button",
  "action_type": "click",
  "element": "optional_button",
  "continue_on_error": true  // 即使失败也继续
}
```

#### 执行逻辑
```python
for flow_step in flow_steps:
    result = await execute_step(flow_step.step)

    if result == FAILED:
        if flow_step.step.continue_on_error:
            # 记录错误但继续
            log_error("Step failed but continuing...")
            continue
        else:
            # 立即停止
            return FAILED
```

#### 使用场景
```
场景1: 关键步骤 (continue_on_error=false)
  登录 → 输入用户名 → 输入密码 → 点击登录
  如果登录失败，后续步骤无意义，应停止

场景2: 可选步骤 (continue_on_error=true)
  打开应用 → 关闭弹窗 → 进入首页
  如果弹窗不存在，不影响后续步骤，应继续
```

### 5. Setup-Main-Teardown 设计

#### 理念
**清晰分离准备、执行、清理三个阶段**

#### 三阶段职责

**Setup (前置)**
```
职责: 准备测试环境
特点:
  - 失败则跳过Main
  - 保证测试环境一致
  - 可复用多个用例

示例:
  - 打开应用
  - 登录系统
  - 进入测试页面
  - 准备测试数据
```

**Main (主流程)**
```
职责: 执行核心测试逻辑
特点:
  - 可以有多个Main流程
  - 是测试的核心部分
  - 结果决定用例成败

示例:
  - 执行业务操作
  - 验证功能点
  - 检查数据正确性
```

**Teardown (清理)**
```
职责: 恢复环境状态
特点:
  - 无论Main成败都执行
  - 保证环境隔离
  - 清理测试数据

示例:
  - 退出登录
  - 删除测试数据
  - 关闭应用
  - 恢复默认设置
```

#### 执行保证
```python
try:
    # Setup
    for flow in setup_flows:
        result = await execute_flow(flow)
        if result == FAILED:
            return FAILED  # Setup失败，不执行Main

    # Main
    main_result = PASSED
    for flow in main_flows:
        result = await execute_flow(flow)
        if result == FAILED:
            main_result = FAILED
            if not continue_on_error:
                break

finally:
    # Teardown (无论如何都执行)
    for flow in teardown_flows:
        await execute_flow(flow)  # 忽略结果

return main_result
```

### 6. 幂等性设计

#### 理念
**支持重复执行，保证结果一致**

#### 实现方式

**1. 元素/步骤名称唯一性**
```python
# 创建时先检查
element = Element.query.filter_by(name="login_button").first()
if element:
    return element.id  # 复用
else:
    element = Element(name="login_button", ...)
    return element.id  # 新建
```

**2. 状态检查**
```python
# 执行前检查状态
if await driver.is_element_present(element):
    return  # 已经在目标状态，无需操作

await driver.click(element)
```

**3. 清理重执行**
```python
# 每次执行前清理
async def execute_testcase(testcase):
    # 清理之前的数据
    await cleanup_test_data()

    # 执行测试
    result = await run_test(testcase)

    # 再次清理
    await cleanup_test_data()

    return result
```

## 总结

### 核心设计原则

1. **分层复用**
   - Element → Step → Flow → Testcase
   - 每一层都可以独立复用
   - 修改上层不影响下层

2. **灵活组合**
   - FlowStep支持引用步骤或嵌套流程
   - Testcase支持Setup/Main/Teardown三阶段
   - 参数可以传递和覆盖

3. **清晰职责**
   - Element: 描述UI元素
   - Step: 定义原子操作
   - Flow: 编排步骤序列
   - Testcase: 组织完整场景

4. **错误控制**
   - continue_on_error细粒度控制
   - Setup失败则跳过Main
   - Teardown无论如何都执行

5. **可维护性**
   - 名称唯一性保证幂等性
   - 多定位符提高鲁棒性
   - 关联表支持灵活组合

### 最佳实践

1. **Element命名**: 使用业务名称而非技术名称
   - ✅ "login_button"
   - ❌ "btn_id_12345"

2. **Step粒度**: 保持原子性，一个步骤做一件事
   - ✅ "input_username"
   - ❌ "login_and_verify"

3. **Flow复用**: 提取公共流程，避免重复
   - ✅ "common_login_flow"
   - ❌ 每个用例都定义登录步骤

4. **参数化**: 使用变量而非硬编码
   - ✅ "{{username}}"
   - ❌ "test_user"

5. **错误处理**: 合理使用continue_on_error
   - 关键步骤: false (失败则停止)
   - 可选步骤: true (失败继续)

这种设计使TestFlow能够灵活、高效地管理复杂的测试场景，同时保持代码的可维护性和可复用性。
