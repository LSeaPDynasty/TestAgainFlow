## 测试步骤说明

每个测试用例包含以下测试步骤：

1. **启动应用** - 启动被测应用
2. **点击设置按钮** - 进入设置界面
3. **点击具体功能项** - 在设置界面中点击对应的功能项
4. **验证界面打开** - 验证目标界面成功打开
5. **返回上级** - 按返回键返回上级界面
6. **重启应用** - 重启应用，准备下一个测试（后置流程）

## 执行方式

### 方式1: 通过前端界面执行
1. 登录TestFlow前端
2. 进入"测试计划"页面
3. 找到"设置界面功能测试计划"
4. 点击"执行"按钮
5. 选择在线设备
6. 启动测试执行

### 方式2: 通过API执行
```bash
POST /api/v1/test-plans/2/run
{
  "device_id": "your_device_id"
}
```

## 注意事项

1. **前置条件**:
   - 设备已连接并在线
   - 被测应用已安装
   - 应用包名: `com.chcnav.rtg`

2. **后置流程**:
   - 每个测试用例执行完成后会重启应用
   - 重启失败不影响下一个测试（continue_on_error=True）

3. **执行顺序**:
   - 按套件顺序串行执行
   - 每个套件内的测试用例按创建顺序执行

4. **失败处理**:
   - 如果某个功能项无法打开，测试用例会失败
   - 失败后继续执行下一个测试用例
   - 执行完成后可查看执行报告
